Backend testcase API documentation¶

api.v1.backend.test_case_get_box()¶

Gets the bounding box for a test case screenshot.

Route:

POST /test_case/get_bounding_box

Parameters (JSON):

Required: ‘label’, ‘api_key’, ‘screenshot_uuid’

Returns (JSON):

success: True if box is not null

box: the bounding box

message: additional info such as cache hit.

api.v1.backend.test_case_upload()¶

Uploads a screenshot for interactive test case creation. The screenshot needs to be in b64 format.

Route:

POST /test_case/upload_screenshot

Parameters (JSON):

Required: ‘screenshot’, ‘api_key’, ‘label’ (name of the element), ‘test_case_uuid’ (name of the test case)

Optional: ‘source’

Returns: (JSON)

success: True if the screenshot was successfully uploaded, False otherwise

key: The screenshot_uuid.
